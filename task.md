# **Опис завдання**

Розробити інтерактивну міні-гру "Підкинь монетку" (Coin Toss). Гравець має зробити прогноз, яка сторона монети випаде — **Heads** (Орел) або **Tails** (Решка), після чого побачить анімований результат підкидання.

# **Ігрова механіка**

## **Основний ігровий цикл**

1\.	**Вибір гравця:** На екрані відображаються дві кнопки — "Heads" та "Tails". Гравець натискає на одну з них, роблячи свій прогноз.  
2\.	**Запит до API:** Після натискання кнопки відправляється асинхронний запит до mock API для отримання результату.  
3\.	**Анімація монети:** Паралельно з запитом запускається анімація підкидання монети (обертання, переворот тощо).  
4\.	**Відображення результату:** Після отримання відповіді від API монета плавно зупиняється на відповідній стороні (Heads або Tails).  
5\.	**Повідомлення про результат:** Протягом 2-3 секунд на екрані відображається повідомлення про виграш ("You Win\!") або програш ("You Lose\!"), після чого воно зникає.  
6\.	**Новий раунд:** Гравець може зробити новий вибір та почати наступний раунд.

## **Mock API**

Необхідно створити просту емуляцію серверного API з наступними характеристиками:

•   	Рандомізація результату на основі *Math.random()*  
•   	Імітація затримки мережі (наприклад, 500-1500 мс)  
•   	Повернення результату у форматі JSON: *{ result: "heads" | "tails" }*

# **Вимоги до UI**

## **Елементи інтерфейсу**

•   	**Монета:** Візуальне представлення монети з двома сторонами. Можна використовувати примітивні фігури (кола різних кольорів для сторін) або зображення з інтернету.  
•   	**Кнопки вибору:** "Heads" та "Tails" з візуальним зворотнім зв'язком при натисканні/ховері.  
•   	**Повідомлення результату:** Динамічне відображення виграшу/програшу з автоматичним зникненням. (Simple Popup)  
•   	**Стан завантаження:** Індикація процесу під час очікування відповіді API.

## **Адаптивність**

Інтерфейс має коректно відображатися в двох орієнтаціях: **Portrait** (вертикальна) та **Landscape** (горизонтальна). Елементи мають перерозподілятися або масштабуватися для оптимального використання простору екрану.

## **Анімації**

Анімації прості, на розсуд кандидата. Можливі варіанти: обертання монети по осі, підкидання вгору-вниз, flip-анімація, або комбінація. Головне — плавність та візуальна зрозумілість.
